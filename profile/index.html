Sistem Kasir (POS)

Proses transaksi penjualan dengan cepat. Dilengkapi manajemen produk (barang & layanan), pembayaran digital, dan cetak struk — semua dalam satu antarmuka kasir yang intuitif.

Real-time Processing

RabbitMQ untuk notifikasi & background jobs. BullMQ dengan Redis untuk job scheduling dan export transaksi besar — tanpa menghambat response API utama.

Laporan Bisnis Lengkap

Analisis pendapatan, pengeluaran, HPP, komisi, dan laba bersih per outlet maupun per staff. Export Excel otomatis dan pengiriman laporan via email langsung ke pemilik bisnis.

Manajemen Stok

Pantau stok masuk dan keluar secara real-time. Kalkulasi Harga Pokok Penjualan (HPP) otomatis dari setiap log stok masuk untuk akurasi laporan keuangan.

Booking & Antrian

Sistem booking slot untuk layanan dengan manajemen antrian. Pelanggan dapat memesan tanpa registrasi (guest order) untuk kemudahan dan kecepatan transaksi.

Autentikasi & Akses

Sistem autentikasi JWT yang aman dengan role-based access control. Setiap peran — Owner, Kasir, dan Customer — hanya mengakses fitur yang relevan.

Arsitektur

Desain Sistem
yang Modular & Skalabel

Setiap layanan dipisahkan dengan tanggung jawab yang jelas — memungkinkan skalabilitas dan maintainability yang optimal.

Frontend Customer

Next.js · Halaman publik, pencarian UMKM, pemesanan & transaksi

Dashboard

Next.js · Panel Owner & Kasir dengan React Query & shadcn/ui

Backend API

Express.js · RESTful API 46+ endpoint, JWT Auth, Winston Logger

Consumer Workers

Worker services · Email, notifikasi, reminder, & service order

Data Layer

PostgreSQL + Redis + RabbitMQ · ACID transactions & async queues

Separation of Concern

Setiap lapisan — Route, Controller, Service, Repository — dipisahkan dengan tegas. Service tidak pernah langsung mengakses database; semua melalui repository.

Validasi Ketat dengan Zod

Semua request body, query, dan params divalidasi menggunakan Zod di lapisan controller sebelum logika bisnis dijalankan.

Async Worker Queue

Operasi berat seperti export Excel masif dan pengiriman email diproses via BullMQ worker queue agar API utama tetap responsif.
